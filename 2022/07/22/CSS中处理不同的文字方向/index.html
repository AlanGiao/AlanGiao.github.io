CSS中处理不同的文字方向

发表于2022-07-22|更新于2022-07-22|webcss
|阅读量:|评论数:

此为MDN web docs的学习笔记。

有的文字是从从上到下，从左到右写的（参照古代中国）。所以需要考虑对这些文字的排版支持。CSS应对这个问题的方法是用逻辑顺序代替物理顺序。

Note: 逻辑顺序是比较新的CSS特性，因此有的较旧的浏览器可能不支持。

Writing mode

如何指定writing mode

CSS使用 writing-mode属性来指定书写模式。此属性有三个值：

  • horizonal-tb, tb 表示 top-to-bottom.
  • vertical-lr, lr 表示 left-to-right.
  • vertical-rl, rl 表示 right-to-left.

inline size 与 block size

在这种情形下，以前的 width, height 已经不再适用，取而代之的是 inline-size 和 block-size属性。inline-size指定 inline 方向的尺寸， block-size指定 block 方向的尺度。

padding, border, margin 的设置

padding、border、margin的尺度设置也有类似的变化。原来的 top/bottom 等价于 block-start, block-end, inline-start, inline-end. 下面是简单的对应关系。

  • padding-left --> padding-inline-start
  • border-bottom --> border-block-end
  • margin-top --> padding-block-start

Note: 因为 border, margin 都是 block 类型的盒子，所以对应的是 block-start/block-end; padding对应的是 inline 类型的盒子，因此对应 inline-start/ inline-end.
